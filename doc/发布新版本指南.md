# 发布新版本指南

Norn 通过 **GitHub Releases** 分发安装包,并内置**自动更新**。
源码在私有仓库(gitea),编译产物发布到公开仓库 [`duyl328/norn`](https://github.com/duyl328/norn)。

## 一次性准备(每台发版机器只做一次)

1. **GitHub CLI 已登录**

   ```bash
   gh auth login   # 选 GitHub.com → HTTPS → 浏览器授权
   gh auth status  # 确认已登录 duyl328
   ```

2. **持有更新签名私钥** `~/.tauri/norn-updater.key`

   这是自动更新的命脉:更新包必须用它签名,老版本才会接受。
   - 公钥已写死在 `src-tauri/tauri.conf.json` 的 `plugins.updater.pubkey`。
   - **私钥务必单独备份**,丢失则无法再发布能被现有用户更新的版本。
   - 如需在新机器重建密钥,见文末。

## 发布流程(每次发版)

```bash
# 1. 构建:版本号自动 +1,并用私钥签出更新包
TAURI_SIGNING_PRIVATE_KEY="$(cat ~/.tauri/norn-updater.key)" \
TAURI_SIGNING_PRIVATE_KEY_PASSWORD="" \
pnpm tauri build

# 2. 发布:生成 latest.json 并上传到 GitHub Release
pnpm release
```

完成后用户的 app 下次启动会自动检测到新版并提示更新。
可选:写更新说明 → `RELEASE_NOTES="修复了 xxx" pnpm release`。

### 这两步分别做了什么

| 步骤 | 动作 |
| --- | --- |
| `pnpm tauri build` | `scripts/tauri-runner.mjs` 先把三处版本号(package.json / tauri.conf.json / Cargo.toml)的 patch +1,再构建,产出 `dmg` + `Norn.app.tar.gz`(更新包)+ `.sig`(签名) |
| `pnpm release` | `scripts/release.mjs` 读版本号 → 生成 `latest.json`(含签名与下载地址)→ `gh release create vX.Y.Z` 上传 dmg / 更新包 / 签名 / latest.json |

产物路径:`src-tauri/target/release/bundle/`(`dmg/` 与 `macos/`)。

## 自动更新原理

- 客户端读取 `https://github.com/duyl328/norn/releases/latest/download/latest.json`。
- 比对版本号,有新版则下载 `Norn.app.tar.gz`,用公钥验签 → 就地替换 → 重启。
- 更新包不带 macOS 隔离标记,因此自动更新**不会**触发 Gatekeeper 拦截。
- 触发点:应用启动静默检查 + 菜单「检查更新」手动触发(代码见 `src/features/workbench/check-updates.ts`)。

## 注意事项

- **版本只增不减**:更新靠版本号比较,务必单调递增(自增脚本已保证)。
- **首次安装的 Gatekeeper 提示**:未做 Apple 公证($99/年),用户首次用 dmg 安装会提示「无法验证开发者」,需 系统设置 → 隐私与安全性 → 仍要打开。**自动更新不受影响**。
- **目前只发 Apple Silicon (aarch64)**。如需 Intel / 通用包,需在构建时增加 target 并在 `scripts/release.mjs` 的 `platforms` 里补对应条目。
- **构建失败先查依赖**:合并新代码后若 TS 报缺模块,先 `pnpm install`。

## 在新机器重建签名密钥(仅当私钥丢失)

> ⚠️ 重建后公钥会变,**已安装的老版本将无法自动更新**(只能让用户重新下载安装一次)。能备份就别重建。

```bash
node node_modules/@tauri-apps/cli/tauri.js signer generate --ci -p "" -w ~/.tauri/norn-updater.key -f
cat ~/.tauri/norn-updater.key.pub   # 把输出填回 tauri.conf.json 的 plugins.updater.pubkey
```
