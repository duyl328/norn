# 安全策略

## 支持范围

安全修复优先面向最新 GitHub Release 和 `main` 分支。旧版本可能不会单独回溯修复，建议用户及时升级。

## 报告漏洞

请不要通过公开 Issue 披露尚未修复的漏洞。请使用 GitHub 的私密漏洞报告入口：

[私下报告安全漏洞](https://github.com/duyl328/norn/security/advisories/new)

报告中请尽量包含：

- 受影响版本和操作系统
- 漏洞影响与可能的攻击场景
- 可复现步骤或最小示例
- 相关日志、截图或代码位置
- 已知缓解方式（如有）

维护者会尽快确认报告、评估影响并协调修复与披露时间。在修复发布前，请避免公开漏洞细节。

## 安全边界

Norn 会访问本地文件、调用系统 Git CLI，并复用用户现有的 Git 配置、凭据、hook 和签名设置。请仅打开可信项目，不要把来自未知来源的仓库、hook 或脚本视为安全内容。
